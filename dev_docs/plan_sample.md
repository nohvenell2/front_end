# 전체 프론트엔드 프로젝트 기획안

## # 개요
- **목적:** 유저의 게임 플레이 기록을 기반으로 개인화된 스팀 게임을 추천해 주는 웹사이트
- **핵심 기능:**
    - 스팀으로 로그인 (Steam API 연동)
    - 유저의 게임 플레이 정보를 대시보드 형태로 시각화
    - 추천 API와 클라이언트 사이드 연산을 결합한 맞춤형 게임 추천 제공
- **필수 사항:**
    - 사이트 기본 언어는 영어(English)로 설정
    - 비정상적으로 잦은 API 요청 방지를 위해 IP 기반 Rate Limiting(예: 1시간 접속 금지) 적용
    - _페이지 및 데이터 캐싱 전략은 추후 아키텍처 검토를 통해 구체화 예정_
- **기술 스택:** Next.js
## # 로그인 페이지
- 스팀 계정을 통한 간편 로그인 기능 구현 (passport-steam 라이브러리 등 활용)
- **예외 처리 안내:** 유저의 스팀 프로필 공개 설정에 따라 플레이 기록을 가져오지 못할 수 있음을 명시하고, 공개 설정으로 변경하는 가이드라인 제공
- 로그인 및 권한 확인이 완료되면 '유저 정보 페이지'로 리다이렉트
## # 유저 정보 페이지
- 유저가 플레이한 게임 목록(이름, 썸네일 이미지, 누적 플레이 시간 등)을 리스트/그리드 형태로 제공
- **데이터 연동 방식:** API를 통해 받아온 기본 정보를 우선 렌더링하고, 게임 ID를 기반으로 **recommend-api를 호출하여** 해당 게임의 장르와 상위 태그 5개를 가져와 매핑해 줌
- **통계 대시보드:** 상단 영역에 유저의 총 플레이 시간, 보유 게임 수, 선호 장르 및 태그 통계를 한눈에 볼 수 있도록 시각화
- 하단 또는 우측 고정 영역에 '게임 추천 설정하기', '게임 추천 받기' 버튼 배치
- '게임 추천 설정하기' 클릭 시 -> 설정 팝업창 오픈
- '게임 추천 받기' 클릭 시 -> 게임 추천 페이지로 이동
## # 게임 추천 설정 팝업
- 클라이언트 사이드 게임 추천 로직은 `dev_docs/client_side_recommend_logic.md` 문서 참조
- **추천 개수 설정:** 5개 ~ 20개 (기본값: 10개)
- **가중치 세부 조절 (1~10점 스케일):**
    - 유사도 점수, 인기도 점수, 평가 점수, 최신성 점수를 유저가 직접 슬라이더나 인풋으로 조절
    - _(내부 로직)_ 유저가 설정한 1~10점 사이의 값들은 클라이언트 계산 직전에 총합이 1.0이 되도록 자동으로 비율 변환(정규화)되어 적용됨
    - **기본값:** 유사도 4, 인기도 3, 평가 2, 최신성 1
- **배제 필터링(Exclusion) 설정:**
    - 출시일: 지정된 날짜 이전에 출시된 고전 게임 제외 (기본값: 2000-01-01)
    - 최소 리뷰 수: 지정된 리뷰 개수 미만인 게임 제외 (기본값: 100개)
    - 긍정 리뷰 비율: 지정된 비율 미만인 '복합적/부정적' 평가 게임 제외 (기본값: 50%)
- 기본값들은 개발자가 수정하기 쉽도록 환경 변수로 관리
## # 게임 추천 페이지
- Steam API에서 가져온 유저 플레이 정보와 설정창에서 지정한 '배제 필터링 값'을 묶어 recommend-api에 전달, 필터링된 상위 후보군 데이터를 응답받음
- 응답받은 데이터를 바탕으로 클라이언트 사이드 로직(`client_side_recommend_logic.md`)을 태워 최종 랭킹 산출
- 계산된 랭킹 순서대로 추천 게임 카드를 화면에 렌더링
- **추천 이유 시각화 (Explainability):** 유저가 랭킹 결과를 직관적으로 납득할 수 있도록, 4가지 핵심 지표(유사도, 인기도, 평가, 최신성)의 최종 정규화 점수를 **스파이더 차트(방사형 그래프)나 4줄의 프로그레스 바** 형태로 카드 내에 시각화
- 각 추천 게임 카드에 해당 스팀 상점 페이지로 바로 넘어갈 수 있는 외부 링크 버튼(Call to Action) 배치
