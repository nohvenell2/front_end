### 클라이언트 사이드 게임 추천 정규화 및 랭킹 산출 로직

**1. TF-IDF 유사도 점수 ($S_{tfidf}$)**
* **설명:** 서버 API로부터 전달받은 게임별 TF-IDF 유사도 원본 점수를 활용함.
* **처리 방식:** 전달받은 후보군(최대 50개) 중 가장 높은 TF-IDF 점수를 1.0으로 설정하여 나머지 점수를 상대 비율로 스케일링하거나, 원본 값을 그대로 적용함.

**2. 인기도 점수 ($S_{pop}$)**
* **설명:** 리뷰 수의 멱법칙(Power Law) 분포에 따른 편차를 완화하고, 추천된 후보군 내에서의 상대적 인기도를 부각하기 위해 로컬 정규화(Local Normalization)를 수행함. 
* **산출식:**
$$S_{pop} = \frac{\ln(1 + \text{개별 게임의 총 리뷰 수})}{\ln(1 + \text{후보군 내 최대 총 리뷰 수})}$$

**3. 평가 점수 ($S_{rate}$)**
* **설명:** API 응답 객체에 포함된 '긍정 리뷰 비율(Positive Review Ratio)' 데이터를 활용함.
* **처리 방식:** 제공되는 데이터 자체가 0에서 1 사이의 정규화된 비율 수치이므로, 클라이언트 단에서의 별도 연산 없이 해당 값을 $S_{rate}$ 지표로 직접 맵핑하여 사용함.

**4. 최신성 점수 ($S_{time}$)**
* **설명:** 게임 출시일로부터 경과된 일수(Days)에 따라 가중치를 점진적으로 감소시키는 지수 감쇠(Exponential Decay) 함수를 적용함.
* **산출식:**
$$S_{time} = e^{-\lambda \cdot \text{출시일로부터 경과한 일수}}$$
*(단, $\lambda = \frac{\ln(2)}{\text{목표 반감기(일수)}}$ 로, 시스템 설정에 따라 사전 정의된 상수를 사용함)*

**5. 최종 랭킹 점수 산출 (Weighted Scoring)**
* **설명:** 정규화 및 맵핑이 완료된 4가지 지표에 설정된 가중치($w$)를 각각 곱하여 최종 랭킹 점수를 합산함. 산출된 최종 점수를 기준으로 배열을 내림차순 정렬하여 렌더링함.
* **산출식:**
$$\text{Final Score} = w_1 S_{tfidf} + w_2 S_{pop} + w_3 S_{rate} + w_4 S_{time}$$
*(단, $w_1 + w_2 + w_3 + w_4 = 1.0$)*
